# Linux.do SidePeek：在列表页直接右侧预览帖子，不用来回跳转

如果你经常在 `linux.do` 里刷长帖、教程帖、配置帖，应该会很熟悉这种体验：

- 点进一篇帖子
- 看完再返回列表
- 再点下一篇
- 重复很多次

我做了一个小扩展，把这个过程改成了 **“列表页 + 右侧抽屉预览”**。
点击帖子标题后，不离开当前列表页，右边直接展开内容，适合连续浏览、快速筛选和多帖对比。

## 它适合看什么内容

这个扩展特别适合下面这些信息密度高、需要连续筛选的帖子：

- `GitHub / Git` 相关教程、问题排查、仓库分享
- `发布 / Release` 说明、版本更新日志、变更摘要
- `AGENTS.md`、项目协作规范、目录说明、工作流约定
- AI 编程、工具链、脚手架、部署、配置类长帖
- 需要连续看多篇再决定深入读哪一篇的主题列表

## 核心功能

- 点击列表标题后，在右侧抽屉直接预览帖子
- 默认优先使用接口渲染，加载更轻、更适合快速浏览
- 如果接口预览失败，会自动回退到整页模式
- 支持 `上一帖` / `下一帖` 快速切换
- 支持 `Alt + ↑ / Alt + ↓` 快捷浏览上下帖子
- 支持 `仅首帖` / `完整主题`
- 支持 `最新回复优先`
- 支持调整抽屉宽度
- 支持在右上角 `选项` 里随时切换预览方式
- 支持 `新标签打开`，需要完整查看时一键回原帖

## 使用方式

1. 打开 `https://linux.do/`、`/latest`、分类页或标签页
2. 点击任意帖子标题
3. 右侧抽屉会直接展开预览
4. 想切换模式时，点右上角 `选项`
5. 想完整打开原帖时，点 `新标签打开`

## 我自己最常用的场景

### 1. 连续刷技术帖时更顺手
比如连着看 `GitHub`、`Git`、部署、Release、工具链、项目结构相关帖子时，不需要“进去一篇 → 返回列表 → 再进下一篇”。

### 2. 快速筛选长帖值不值得深读
有些帖子标题很吸引人，但真正有价值的信息可能只在首帖或者后面的几条回复里。右侧抽屉可以很快判断要不要完整打开。

### 3. 看规范类内容更方便
像 `AGENTS.md`、协作约定、脚本说明、发布说明、项目目录结构这种内容，很多时候只是想先扫一眼重点，用抽屉看会更轻松。
